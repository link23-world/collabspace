Toolbox

What is a tool?

If it looks like a tool, and solves a problem, then it is a tool.

This could include software, a how-to guide for solving a particular task, a handbook for equity, a reference for deciding how to appropriately use equitable terminology, and more.

For link23, tools must fulfil the following criteria:

  1. The tool must contribute to solving genuine problem. The tool could be a small piece in a large pipeline or an end-to-end solution but it must be built to solve real-world problems. For example, a tool that provides colour palettes that look like artwork would be out-of-scope whereas a tool that provides colour-blind-friendly palettes may be in scope.
  2. The problems they solve must have a demonstrable link to genomic equity. This means the tool could either be a tool for genomic analysis that takes equity into consideration in some way (e.g., analysis tools developed for people from non-European ancestry) or a tool for equity that can be applied to genomics (e.g., a handbook for developing domain-agnostic software that is equitable for users). The link to genomic equity may not be fully developed if there is a clear path for development (e.g., a tool for equitable analysis of health records data that could be adapted for equitable analysis of genomic data).

So without anymore preamble... the link23 toolbox!
